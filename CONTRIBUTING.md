## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/xrate.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `pip install -e '.[dev]'`
5. Make your changes
6. Run tests: `make test` or `pytest`
7. Run linters: `make lint`
8. Commit your changes: `git commit -m "Add: your feature description"`
9. Push to your fork: `git push origin feature/your-feature-name`
10. Open a Pull Request

## Code Style

- Follow PEP 8 guidelines
- Use type hints where appropriate
- Add docstrings to functions and classes
- Keep functions focused and small
- Write meaningful commit messages

## Testing

- Write tests for new features
- Ensure all existing tests pass
- Aim for good test coverage
- Test edge cases

## Commit Messages

Use conventional commit format:
- `Add:` for new features
- `Fix:` for bug fixes
- `Update:` for updates/enhancements
- `Refactor:` for code refactoring
- `Docs:` for documentation changes

Example:
```
Add: support for new API provider

- Implemented new provider adapter
- Added tests for provider
- Updated documentation
```

## Pull Request Process

1. Update README.md if needed
2. Add/update tests if needed
3. Ensure CI passes
4. Get at least one review before merging

## Questions?

Feel free to open an issue for questions or discussions!
